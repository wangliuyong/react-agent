# 灵犀 使用手册

跨平台桌面 AI 助手（Electron + React），支持 **智能对话、多渠道发布、流程编排、AI/Remotion 视频、定时任务与消息通知**。  
大模型默认对接 **阿里云百炼（DashScope）** OpenAI 兼容接口；数据全部缓存在本机，不使用数据库。

---

## 1. 环境要求

- Node.js ≥ 20（推荐 `nvm use 20`）
- pnpm ≥ 9
- macOS / Windows / Linux

## 2. 安装与启动

```bash
cd react-agent
pnpm install
pnpm install:browser          # 经 npmmirror 加速下载 Playwright Chromium
pnpm install:remotion-browser # Remotion 渲染依赖（使用 Remotion 视频时需要）
pnpm dev
```

`pnpm install:browser` 默认走国内镜像（npmmirror）。若镜像异常，可用官方源：`pnpm install:browser:official`。

## 3. 配置模型与 API

1. 打开 [阿里云百炼控制台](https://bailian.console.aliyun.com/) 创建 API Key。  
2. 启动应用后，侧栏底部进入 **设置** → **模型与 API**。  
3. 填写：
   - **DASHSCOPE API Key**
   - **Base URL**（默认 `https://dashscope.aliyuncs.com/compatible-mode/v1`）
   - **模型**（默认 `qwen-plus`，可改为 `qwen-max` / `qwen-turbo` 等）
4. 点击保存。

> Key 仅保存在 Electron `userData/react-agent-data/settings.json`，不会提交到 git。

### 多模型连接（可选）

在 **设置 → 多模型连接** 中可配置多条 `ModelConnection`（不同供应商、模型、能力标签），并为各 Agent 角色（general、researcher、writer、publisher、scriptwriter 等）指定默认连接。Agent 会按任务内容自动推断能力（vision、reasoning、longContext 等）并路由到合适模型。

### 思考模式

在 **模型与 API** Tab 可开关 **思考模式**。开启后，聊天界面会流式展示模型推理过程（`thinking_delta`），推理结束后再展示工具调用与正式回答。

## 4. 界面导航

### 灵犀 AI 助手模式（默认）

| 入口 | 说明 |
|------|------|
| **+ 新任务** | 创建新会话，进入聊天页 |
| **技能市场** | 浏览、启用/禁用、导入、编辑领域技能 |
| **规则** | 管理 MDC 规则，启用后注入 Agent 提示 |
| **流程** | 可视化工作流列表与画布编辑 |
| **渠道** | 发布渠道与通知渠道配置、登录、测试 |
| **发布** | 发布工作台（多子任务或串联流程） |
| **定时任务** | 定时规则、立即执行、执行记录 |
| **历史对话** | 助手模式下的会话列表 |
| **设置**（底部图标） | 模型、工具、渠道状态、资产等 |

聊天页不在侧栏导航中，通过 **+ 新任务** 或点击历史会话进入。

### 业务系统模式

在聊天页顶栏 Segmented 切换到 **业务系统**：

- 侧栏仅显示 **历史对话**
- 主区为全量会话管理，可查看工作流 context、节点入参/出参、通知 HTTP 调试信息
- 顶栏可切回 **灵犀 AI 助手**

## 5. 聊天与任务

### 5.1 快捷任务卡

新会话欢迎页展示 8 张快捷任务卡（如「发一条小红书」「一句话生成视频」「Remotion 短片成片」等），点击即可发送预设指令。可点 **换一批** 轮换更多卡片。

### 5.2 聊天执行指令

无需进入对应页面，直接在聊天中发送：

| 格式 | 动作 |
|------|------|
| `执行定时任务：名称` / `运行定时任务 名称` | 匹配并执行定时任务 |
| `执行任务：名称` / `运行任务 名称` | 匹配并运行发布计划 |
| `执行流程：名称` / `运行流程 名称` | 匹配并运行工作流（publish 镜像流程请用「执行任务」） |

### 5.3 发一条小红书（推荐路径）

1. 点击 **+ 新任务**。  
2. 输入例如：

   > 帮我发一条小红书，内容关于今日 A 股收盘速读，标题不超过 20 字。请从相关新闻网页抓取配图后发布。

3. Agent 会优先调用 `fetch_web_images` 从来源页下载配图。  
4. **本地上传为可选**：需要指定某张图时，再点输入框旁回形针选择图片。  
5. 右侧会打开 **智能体浏览器**，进入创作台；未登录时按提示扫码后点 **继续**。  
6. 非「完全访问」模式下，正式点「发布」前会再次确认。

### 5.4 配图优先级

1. `imagePaths` / `fetch_web_images` 下载的本地文件  
2. `imageSourceUrl` / `imageUrls`（工具内自动下载）  
3. 用户本轮上传的附件（可选）

### 5.5 聊天界面功能

| 功能 | 说明 |
|------|------|
| 流式 Markdown | 实时渲染 Agent 回复 |
| 思考过程 | 独立折叠区展示推理（受思考模式开关控制） |
| 工具调用 | 折叠展示工具参数与结果 |
| 代码块 | 超过 12 行默认折叠，支持语法高亮 |
| HTML 预览 | 生成的 HTML 可在聊天内 iframe 预览，或「在浏览器打开」 |
| K 线图表 | A 股分析结果内嵌 ECharts 图表 |
| 工具进度 | Remotion 渲染等长耗时操作展示进度条 |
| 产物操作 | 打开文件所在目录（访达/资源管理器）、浏览器打开 HTML |
| 任务清单 | 浮动清单跟踪步骤；完成后可「总结为技能」 |
| 智能体浏览器 | 顶栏地球图标开关右侧截帧预览 |
| 附件上传 | 输入框回形针；vision 能力可识图 |

### 5.6 权限说明

| 模式 | 行为 |
|------|------|
| 需确认（默认） | 登录、正式发布、Remotion 渲染等敏感操作前暂停等人 |
| 完全访问 | 跳过部分确认（仍建议人工看一眼发布结果） |

## 6. AI 视频与 Remotion

### AI 叙事视频管线

适合「一句话生成视频」类任务。Agent 依次调用：

1. `generate_script` — 生成剧本  
2. `generate_storyboard` — 生成分镜  
3. `generate_scene_assets` — 生成场景素材（图/视频/配音）  
4. `compose_video` — 合成成片  

成片保存在 `videos/` 目录，可在 **设置 → 资产** 中预览。

### Remotion 程序化视频

适合精确动效、字幕、数据可视化、品牌模板类视频：

1. `remotion_init_project` — 初始化 React 视频工程  
2. `write_file` — 编写 Composition 代码  
3. `remotion_studio`（可选）— 本地预览  
4. `remotion_render` — 导出 mp4（渲染前需确认，界面展示渲染进度）

首次使用 Remotion 渲染前请执行 `pnpm install:remotion-browser`。

## 7. 发布工作台

1. 侧栏进入 **发布**。  
2. 可点 **导入示例** 得到多渠道示例子任务。  
3. 编辑子任务的主题与内容说明，或选择串联流程模式。  
4. 点 **运行**：会新建主聊天会话，由 Agent **串行**执行子任务。  
5. 执行进度请在主聊天窗口与任务清单中查看。

---

## 8. 流程编排

1. 侧栏进入 **流程**。  
2. 新建工作流，在可视化画布中拖拽节点（Agent、工具、通知、条件、并行、输入/输出等）。  
3. 保存后点击 **运行** 立即执行。  
4. 也可在定时任务或发布计划中关联流程。

---

## 9. 定时任务

1. 侧栏进入 **定时任务**。  
2. 点击 **新建任务**，配置：
   - **重复规则**：单次 / 每天 / 每周
   - **执行时间**：具体时刻或星期几
   - **动作类型**：发布计划 / 自定义指令 / 流程
   - **通知渠道**（可选）：执行完成后推送消息
3. 保存后由主进程每 30 秒轮询调度。  
4. 可点击 **立即执行** 手动触发；执行记录会关联到对应聊天会话。

---

## 10. 技能市场与规则

### 技能市场

1. 侧栏进入 **技能市场**。  
2. 浏览内置技能（小红书发布、Remotion 套件、AI 视频等）。  
3. 启用/禁用技能；支持 **从模板安装**、**从链接导入**、**手动新建/编辑**。  
4. 任务完成后可在任务清单点 **总结为技能**，将成功步骤保存为新技能。  
5. 已启用技能的摘要注入 Agent 系统提示；运行时 `use_skill` 按需加载完整内容。

### 规则

1. 侧栏进入 **规则**。  
2. 管理 MDC 规则文件（如「简体中文回复」默认规则）。  
3. 启用后注入各角色 system prompt。

## 11. 渠道管理

### 发布渠道

- **小红书**、**抖音**（已启用）
- **视频号**（占位，即将上线）

在 **渠道** 页配置渠道参数、打开登录页、检测登录态。可在渠道上开关「拟人操作」。

### 通知渠道

- **飞书**、**Webhook**（可配置并测试）
- **微信通知**、**QQ 通知**（占位）

Agent 通过 `notify_message` 工具推送消息；定时任务也可绑定通知渠道。

## 13. 本地数据目录

应用数据位于系统 userData 下的 `react-agent-data/`：

| 路径 | 内容 |
|------|------|
| `settings.json` | API Key、多模型连接、角色映射、权限 |
| `sessions/` | 聊天会话 |
| `publish-plans/` | 发布计划 |
| `scheduled-tasks/` | 定时任务 |
| `workflows.json` | 流程定义 |
| `workflow-runs.json` | 流程运行记录 |
| `channels.json` | 渠道配置 |
| `skill-states.json` | 技能启用状态 |
| `browser-profile/` | 有头浏览器登录 Cookie |
| `browser-profile-headless/` | 无头抓取专用 Profile |
| `artifacts/` | Agent 通用产出 |
| `videos/` | 视频成片与分镜素材 |
| `resources/` | 安装版可写技能/规则副本 |

macOS 典型路径：`~/Library/Application Support/lingxifa/react-agent-data/`

清除登录：**设置 → 渠道状态** → **清除全部发布渠道登录态**。

## 16. 常见问题

**Q: 提示未配置 API Key**  
A: 先到设置页 → 模型与 API 保存 Key。

**Q: 上传控件找不到 / 填不了标题**  
A: 平台 DOM 可能改版。Agent 会回退到 `browser_snapshot` + 原子工具；也可在右侧浏览器手动补全后点继续。

**Q: Chromium 弹窗和右侧预览同时出现？**  
A: 当前为有头 Playwright 便于登录与调试，右侧同步截帧，属预期行为。

**Q: 提示「正在现有的浏览器会话中打开」或 browser has been closed？**  
A: 多为上次异常退出留下的 Profile 锁。应用会在启动浏览器前自动清理 `SingletonLock`；请重启 `pnpm dev` 后重试。若仍失败，可到设置页清除发布渠道登录态。

**Q: Remotion 渲染失败或缺少浏览器？**  
A: 执行 `pnpm install:remotion-browser` 安装 Remotion 渲染依赖。

**Q: 如何在聊天里直接执行任务？**  
A: 发送「执行定时任务：xxx」「执行任务：xxx」或「执行流程：xxx」，名称需与列表中的任务/计划/流程名称匹配。

**Q: 文档放哪？**  
A: 项目文档统一放在本仓库 [`doc/`](./) 目录：使用手册、项目架构等；概览见根目录 [README.md](../README.md)。
