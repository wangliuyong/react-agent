# 灵犀 项目架构

本文档描述灵犀（react-agent）的整体技术架构、模块划分与扩展方式。面向需要二次开发或排查问题的开发者。

---

## 1. 架构总览

灵犀是典型的 **Electron 三进程桌面应用**：主进程负责 Agent 推理、工具执行、浏览器自动化与工作流引擎；渲染进程负责 React UI；preload 脚本通过 `contextBridge` 暴露安全的 IPC API。

```
┌──────────────────────────────────────────────────────────────────────┐
│                         渲染进程 (React)                              │
│  AppShell ─ features/chat|business|publish|schedule|skills|rules|    │
│             channels|workflows|settings|browser                       │
│       │                                                              │
│       │  window.api.* (preload)                                      │
└───────┼──────────────────────────────────────────────────────────────┘
        │ IPC invoke / event push
┌───────┼──────────────────────────────────────────────────────────────┐
│       ▼              主进程 (Node.js)                                 │
│  ipc.ts ── store/* (JSON 持久化)                                     │
│       │                                                              │
│       ├── agent/                                                     │
│       │    ├── graph-bridge.ts   # 聊天/步骤入口；LangGraph ↔ Event  │
│       │    ├── graph/            # chat 多智能体 + ReAct 子图         │
│       │    ├── model-router.ts   # 能力推断与连接路由                │
│       │    ├── llm-langchain.ts  # ChatOpenAI → 多供应商             │
│       │    └── tools/*           # AgentTool 注册表 + LC 适配          │
│       │                                                              │
│       ├── workflow/                                                  │
│       │    ├── compile-to-langgraph.ts  # Definition → StateGraph    │
│       │    └── engine.ts                # Run 生命周期 + 节点执行复用 │
│       │                                                              │
│       ├── browser/service.ts     # Playwright 有头/无头               │
│       ├── media/remotion-service.ts  # Remotion 工程与渲染             │
│       ├── notify/                # 通知渠道发送                        │
│       └── schedule/scheduler.ts  # 定时轮询                            │
└──────────────────────────────────────────────────────────────────────┘
        │
        ▼
  userData/react-agent-data/   （settings、sessions、plans、profile…）
```

### 设计原则

| 原则 | 实现 |
|------|------|
| LangGraph 编排 | 聊天 Supervisor 多角色 + 工作流 `compile-to-langgraph` |
| 工具即能力边界 | 新增功能优先注册 `AgentTool`，经适配器进 LangChain |
| 事件兼容 | 推送 `event:agent`（text_delta / tool_* / tool_progress / await_user / done 等） |
| 无数据库 | 全部数据 JSON 落盘到 `userData`；图 checkpoint 进程内 MemorySaver |
| 契约共享 | `shared/types.ts` 定义 IPC 通道与实体类型，主/渲染两端共用 |
| 按业务域拆分 | 前端 `src/features/*`，每个 feature 含 components / hooks / api |

## 3. 进程通信（IPC）

### 3.1 通道命名约定

定义于 `shared/types.ts` 的 `IpcChannels`：

- **读操作**：`query:*`（如 `query:settings`、`query:sessions`、`query:workflows`）
- **写操作**：`post:*`（如 `post:settings`、`post:agent:chat`、`post:run-workflow`）
- **事件推送**（主 → 渲染）：`event:*`（如 `event:agent`、`event:browser-frame`）

### 3.2 Preload 桥接

`electron/preload/index.ts` 通过 `contextBridge.exposeInMainWorld('api', …)` 暴露 `ElectronApi` 接口。渲染进程只通过 `window.api` 访问主进程。

### 3.3 事件流

Agent 执行过程中的状态通过 `webContents.send('event:agent', event)` 推送到渲染进程：

```typescript
type AgentEvent =
  | { type: 'text_delta'; ... }         // 流式文本
  | { type: 'thinking_delta'; ... }     // 推理过程增量
  | { type: 'thinking_complete'; ... }  // 推理结束
  | { type: 'message'; ... }            // 完整消息落盘
  | { type: 'tool_start'; ... }         // 工具开始
  | { type: 'tool_result'; ... }       // 工具结果
  | { type: 'tool_progress'; ... }      // 长耗时工具进度（如 Remotion 渲染）
  | { type: 'task_update'; ... }        // 任务清单更新
  | { type: 'await_user'; ... }         // 等待用户（登录/确认/方案选择）
  | { type: 'browser_open'; ... }       // 浏览器导航
  | { type: 'agent_role'; ... }         // 当前 Agent 角色切换
  | { type: 'model_switch'; ... }       // 模型连接切换
  | { type: 'token_update'; ... }       // Token 用量更新
  | { type: 'workflow_toast'; ... }     // 工作流 Toast 节点
  | { type: 'session_started'; ... }    // 新会话创建（任务/流程触发）
  | { type: 'done'; ... }               // 本轮结束
  | { type: 'error'; ... }              // 错误
```

渲染进程在 `useSessionStore.bindAgentEvents()` 中订阅并更新 UI。

## 4. Agent 编排（LangGraph）

核心文件：`electron/main/agent/graph-bridge.ts`、`graph/chat-graph.ts`

### 4.1 聊天多智能体图

```
START → supervisor → general | publish管线 | video管线 → END

publish: researcher → writer → publisher
video:   scriptwriter → videographer → editor
```

- Checkpointer：进程内 `MemorySaver`，`thread_id = sessionId`
- 各角色走受限 ReAct 子图 + 工具白名单（`role-tools.ts`）
- Supervisor 无工具，只输出 JSON 路由决策；解析失败有关键词兜底（`model-router.ts`）

### 4.2 模型路由

`model-router.ts` + `llm-langchain.ts`：

| 能力 | 说明 |
|------|------|
| `chat` | 通用对话 |
| `reasoning` | 复杂推理 |
| `vision` | 含附件/图片时自动推断 |
| `longContext` | 超长上下文（>12k 字） |
| `creative` | 创意写作 |

解析顺序：显式 capability → `roleModelMap[role]` → `defaultConnectionId`

### 4.3 系统提示与技能注入

- 角色提示在 `graph/prompts.ts`
- 构图时 `buildRoleSystemPrompt()` 注入：角色 prompt + 用户规则 + **技能目录摘要**（按角色字符预算截断）
- 运行时 `use_skill` 读取完整 `SKILL.md`
- Supervisor **不**加载规则/技能目录

### 4.4 会话控制

| 机制 | 说明 |
|------|------|
| `abortMap`（graph-bridge） | 每会话一个 `AbortController`，IPC `postAgentAbort` |
| LangGraph `interrupt` | 登录/确认等待；IPC `postAgentContinue` → `Command({ resume })` |
| `maxTurns` | 设置页可配，防止无限工具循环 |

## 5. 工具注册表

定义于 `electron/main/agent/tools/`，通过 `getAllTools()` 统一注册。设置页通过 `catalog.ts` 的 `queryAgentToolsCatalog()` 提供只读浏览与源码预览。

### 5.1 当前工具列表

| 分类 | 工具名 | 用途 |
|------|--------|------|
| 技能/模型 | `use_skill` | 按需加载完整技能文档 |
| | `switch_model` | 动态切换模型连接 |
| 文件 | `list_attachments` / `read_file` / `write_file` | 附件与 artifacts 读写 |
| 任务 | `update_task_list` | 更新聊天区任务清单 |
| | `present_plan_choices` | 多方案让用户选择 |
| 图片 | `generate_image` | 文生图（万相） |
| | `fetch_web_images` | 从网页下载配图 |
| 数据 | `fetch_hot_topics` | 科技/AI 热点 |
| | `query_weather` | 城市天气 |
| | `query_ashare_kline` / `query_ashare_realtime_analysis` | A 股行情与分析 |
| | `query_web_data` | 联网调研 |
| AI 视频 | `generate_script` / `generate_storyboard` | 剧本与分镜 |
| | `generate_scene_assets` / `compose_video` | 场景素材与合成 |
| Remotion | `remotion_init_project` / `remotion_studio` / `remotion_render` | 程序化视频工程与渲染 |
| 浏览器 | `browser_navigate` / `browser_snapshot` / `browser_click` / `browser_type` / `browser_upload` / `browser_wait` | Playwright 原子操作 |
| 发布 | `xhs_publish_note` / `douyin_publish_note` | 小红书/抖音图文发布 |
| 通知 | `notify_message` | 飞书/Webhook 等消息推送 |

### 5.2 工具接口

```typescript
interface AgentTool {
  name: string
  description: string
  parameters: JSONSchema
  permission?: 'normal' | 'sensitive'  // sensitive 需用户确认
  execute(args: unknown, ctx: ToolContext): Promise<ToolResult>
}

interface ToolContext {
  sessionId: string
  emit: (event: AgentEvent) => void
  settings: AppSettings
  postActiveCapability?: (cap: ModelCapability) => void
}
```

`ToolResult` 可包含 `awaitUser: true`，loop 会 emit `await_user` 并挂起直到用户继续。长耗时工具可通过 `ctx.emit({ type: 'tool_progress', ... })` 推送进度。

### 5.3 角色工具白名单

`graph/role-tools.ts` 定义各角色可用工具：

| 角色 | 工具策略 |
|------|----------|
| `supervisor` | 无工具 |
| `general` | 全部工具 |
| `researcher` / `writer` / `publisher` 等 | 白名单子集 |

### 5.4 扩展新工具

1. 在 `electron/main/agent/tools/` 新建模块，实现 `AgentTool`。
2. 在 `tools/index.ts` 的 `getAllTools()` 中追加。
3. 若需新渠道，同步更新 `shared/publish-channels.ts` 并实现对应 `*-publish.ts`。
4. 在 `resources/skills/` 添加领域技能文档（可选）。

## 6. Remotion 视频服务

核心文件：`electron/main/media/remotion-service.ts`、`agent/tools/remotion-tools.ts`

### 6.1 流程

```
use_skill('react-agent-remotion')
    → remotion_init_project（复制 resources/remotion/starter）
    → write_file（编写 Composition）
```

### 6.2 与 AI 视频管线的区别

| 维度 | AI 视频管线 | Remotion |
|------|-------------|----------|
| 入口工具 | `generate_script` 等 | `remotion_init_project` |
| 产出方式 | 万相 T2I/I2V + TTS 合成 | React 代码 + Remotion 渲染 |
| 适用场景 | 叙事短片、一键成片 | 精确动效、字幕、图表、品牌模板 |

## 7. 浏览器服务

核心文件：`electron/main/browser/service.ts`

### 7.1 设计要点

- **双 Profile**：`browser-profile/`（有头发布/登录）与 `browser-profile-headless/`（无头抓取）隔离，避免 SingletonLock 冲突。
- **有头模式**：`headless: false`，便于用户扫码登录与人工介入。
- **截帧推送**：定时 `page.screenshot()` → `event:browser-frame` → 渲染进程预览面板。
- **锁清理**：启动前 `releaseBrowserProfileLock()` 清理残留 Chrome 进程。
- **拟人交互**：`human-input.ts` 模拟真实鼠标移动与打字间隔。

### 7.2 发布脚本分层

```
xhs_publish_note (工具入口)
    └── xhs-publish.ts (业务流程)
            └── xhs-dom.ts (DOM 选择器)
            └── service.ts (底层 page 操作)
            └── human-input.ts (拟人点击/打字/上传)
```

## 10. 前端架构

### 10.1 视图路由

`stores/app-store.ts` 维护 `view: AppView`：

```typescript
type AppView =
  | 'chat' | 'business' | 'publish' | 'settings'
  | 'skills' | 'rules' | 'channels' | 'schedule' | 'workflows'
```

视图持久化到 `localStorage`（key: `lingxi:app-view`）。`AppShell` 根据 `view` 渲染对应 feature 页面。

### 10.2 Feature 模块结构

以 `src/features/chat` 为例：

```
chat/
├── index.ts              # 对外导出
├── types.ts              # feature 局部类型
├── api.ts                # window.api 封装
├── hooks/
│   └── useSessionStore.ts  # Zustand store

    ├── ChatPage/
    ├── ChatInput/
    ├── MessageList/

    └── WelcomeHero/      # 快捷任务卡
```

各 feature 遵循相同模式：`api.ts` 封装 IPC → `hooks/*Store.ts` 管理状态 → `components/` 纯 UI。

### 10.3 状态管理

使用 **Zustand** 按域拆分 store：

| Store | 职责 |
|-------|------|
| `useAppStore` | 当前视图、侧栏折叠 |
| `useSessionStore` | 会话列表、消息、Agent 运行状态 |
| `useSettingsStore` | 应用设置 |
| `usePublishStore` | 发布计划 CRUD |
| `useScheduleStore` | 定时任务 CRUD + 调度事件 |
| `useSkillsStore` | 项目技能 CRUD |
| `useBusinessStore` | 业务系统菜单与追溯 |
| `useBrowserControl` | 浏览器预览状态 |

`App.tsx` 在 `useEffect` 中统一 hydrate 并订阅 Agent / Schedule 事件。

## 11. 发布与调度

### 11.1 发布渠道注册表

`shared/publish-channels.ts` 统一管理渠道元数据：

```typescript
interface PublishChannelMeta {
  id: PublishChannelId       // 'xhs' | 'douyin' | 'wechat_channels'
  label: string
  enabled: boolean
  publishTool: string
  titleMaxLength?: number
  agentHint: string
}
```

### 11.2 发布计划 → Agent 指令

`shared/publish-prompt.ts` 的 `buildPublishPlanPrompt(plan)` 将工作台计划转为结构化 Agent 指令。

### 11.3 定时调度器

`electron/main/schedule/scheduler.ts`：

- 应用启动时 `startScheduleService()`，每 30 秒 `tick()`。
- 检查 `enabled` 且 `nextRunAt <= now` 的任务。
- 动作类型：发布计划 / 自定义指令 / 流程。
- 创建 `type: 'schedule'` 会话，调用 `runAgentChat()`。
- `agent-hook.ts` 监听 `done` / `error` 事件，更新 `lastRunStatus` 与 `nextRunAt`。

### 11.4 聊天执行指令

渲染进程 `parseChatExecutionCommand()` 解析用户消息中的「执行定时任务/任务/流程 + 名称」，匹配对应实体并触发执行。

## 12. 技能与规则系统

### 12.1 技能

技能文件位于仓库 `resources/skills/<skill-id>/SKILL.md`。开发环境直接维护该目录；安装版首次启动将内置资源补入 `userData/react-agent-data/resources`。

| 能力 | 实现 |
|------|------|
| 列表/详情 | `store/skills.ts` 扫描目录 |
| 启用/禁用 | `skill-states.json` 记录开关 |
| 注入提示 | `queryEnabledSkillPrompt()` 拼接已启用技能摘要 |
| 运行时加载 | `use_skill` 工具读取完整正文 |
| 链接导入 | `store/skill-import.ts` |
| 会话总结 | `store/skill-summarize.ts` |

### 12.2 规则

规则维护使用 `resources/rules/*.mdc`，启停状态写入 frontmatter `alwaysApply`；旧版 `rules.json` 仅在启动时迁移。

## 13. 通知系统

`electron/main/notify/` 实现多渠道消息发送：

- **飞书**：text / post / image / share_chat
- **Webhook**：通用 HTTP POST

Agent 通过 `notify_message` 工具调用；定时任务与发布计划可绑定通知渠道。工作流 `notify` 节点复用同一发送逻辑。

## 14. 构建与别名

`electron.vite.config.ts` 配置三端构建：

| 端 | 入口 | 说明 |
|----|------|------|
| main | `electron/main/index.ts` | Node 环境，externalize 依赖 |
| preload | `electron/preload/index.ts` | 桥接脚本 |
| renderer | `src/index.html` | React SPA |

渲染进程别名：

- `@` → `src/`
- `@shared` → `shared/`
