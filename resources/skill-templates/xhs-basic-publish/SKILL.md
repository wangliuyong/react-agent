# 小红书基础发布

## 适用场景

用户已给出明确主题，无需热搜调研，例如：

- 「帮我发一条小红书，内容关于今日 A 股收盘速读，标题不超过 20 字」
- 「从相关新闻网页抓取配图后发布」

## 标准任务清单（update_task_list）

| id | 标题 |
|----|------|
| 1 | 搜索主题相关新闻 |
| 2 | 撰写标题与正文 |
| 3 | 从来源页抓取配图 |
| 4 | 发布到小红书 |

## 执行步骤

### 1. 搜索新闻

`browser_navigate` 打开搜索引擎或新闻站，搜索用户主题，`browser_snapshot` 阅读并选定来源页。

### 2. 撰写内容

- 标题 ≤20 字
- 正文口语化，适合小红书风格

### 3. 抓取配图

```json
{ "pageUrl": "https://...来源页", "maxCount": 3 }
```

或在 `xhs_publish_note` 中传 `imageSourceUrl` 让工具内下载。

### 4. 发布

```json
{
  "title": "...",
  "content": "...",
  "imagePaths": [...],
  "autoPublish": true
}
```

## 配图优先级

1. `fetch_web_images` 下载的 `imagePaths`
2. `imageSourceUrl` / `imageUrls`
3. 用户本轮附件（可选）

缺图时返回明确提示，不要默认要求用户上传。

## 注意

- 未登录时暂停等人扫码
- 非完全访问模式下正式发布前会再次确认
- 以工具返回为准，不要编造已发布成功
