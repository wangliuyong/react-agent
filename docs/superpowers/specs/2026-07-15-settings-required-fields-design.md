# 聊天配置守卫与渠道状态设计

## 目标

- 聊天页加载时检测 API Key 与 Base URL；缺少任一配置时提示用户并跳转设置页。
- 设置页集中展示发布渠道登录状态与通知渠道配置状态。
- 浏览器 Profile 继续负责持久化各发布渠道 Cookie；状态检测结果不额外落盘。

## 范围

- 在聊天页增加设置加载完成后的配置守卫。
- 设置页进入时自动检测已接入发布渠道登录状态。
- 展示全部发布渠道与通知渠道，其中发布渠道显示登录状态，通知渠道显示配置状态。
- 抽取设置页渠道状态区的状态查询与展示逻辑，复用现有渠道 API、渠道 Store 和 Ant Design 组件。
- 不修改主进程登录检测算法与浏览器 Profile 存储方式。

## 交互与错误处理

- 设置尚未加载完成时不执行跳转，避免默认空 API Key 造成误判。
- API Key 或 Base URL 去除首尾空格后为空时，显示缺失项提示并切换到设置页。
- Base URL 为空时提示“请填写 Base URL”。
- 模型为空时提示“请选择模型”。
- 校验失败时不调用 `postSettings`，也不展示“已保存”。
- 设置页挂载时自动检测发布渠道登录态，并提供手动重新检测。
- 发布渠道提供“打开登录页”；登录操作后重新检测。
- 通知渠道显示“已配置”“未配置”或“即将上线”。
- 登录检测失败时展示错误状态和提示，不阻塞设置表单。
- 清除浏览器 Profile 后立即刷新发布渠道状态。

## 验证

- 按用户要求不新增自动化测试。
- 运行 TypeScript 类型检查与生产构建。
